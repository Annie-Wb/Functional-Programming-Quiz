# Quiz 03 - Reactive Programming

Exercises

1. Reactive ideas
   - Explain what reactive programming means.
   - Compare it with ordinary event-driven programming.

2. Signals and events
   - Describe the difference between a signal and an event stream.
   - Give a small example of a time-varying value.

3. Functional reactive programming
   - Explain why FRP fits naturally with functional programming.
   - Outline how you would model a changing UI state.

4. Design discussion
   - Give one advantage of reactive style over manual callback handling.
   - Give one drawback or challenge of FRP.
